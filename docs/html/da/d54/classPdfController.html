Description détaillée

Gère la génération, le remplissage et l'affichage des documents PDF liés aux rapports et attestations.

Ce contrôleur centralise toutes les opérations relatives aux fichiers PDF :

  • Génération dynamique à partir de modèles prédéfinis.
  • Remplissage automatique de formulaires (Cerfa, attestations TVA, bulletins d'intervention...).
  • Téléchargement et affichage sécurisés des documents liés à un token.
  • Upload et gestion des devis PDF certifiés.

◆ generateAttestationTVA()

App::Http::Controllers::PdfController::generateAttestationTVA ( class Request $request)

Génère une attestation TVA personnalisée pour les travaux de rénovation.

Cette méthode gère la génération d'attestations TVA avec des fonctionnalités avancées :

  • Support des requêtes JSON (POST) et paramètres URL (GET).
  • Remplissage automatique des champs d'identité (nom, prénom, adresse).
  • Gestion des cases à cocher pour le type de logement et l'affectation.
  • Traitement des différents types de travaux avec leurs détails.
  • Intégration de signatures numériques en base64.
  • Formatage des dates au format français (d/m/Y).
Paramètres
\Illuminate\Http\Request$requestRequête HTTP contenant soit :
  • JSON (POST) : uid, document, client + données du formulaire
  • Query params (GET) : uid, document, client
Renvoie
\Symfony\Component\HttpFoundation\BinaryFileResponse|\Illuminate\Http\JsonResponse Fichier PDF généré en affichage inline ou erreur JSON.
Exceptions

Illuminate\Http\Exceptions\HttpResponseException Si le fichier JSON de données n'existe pas.

Exceptions

InvalidArgumentException Si les paramètres requis sont manquants.

Note
La signature est automatiquement convertie de base64 vers PNG et intégrée au PDF.
Les coordonnées de positionnement sont spécifiquement calibrées pour le modèle d'attestation TVA.
Avertissement
Nécessite un fichier JSON préalablement créé contenant toutes les données du formulaire.
Voir également
generateDownloadPDF() Pour la création initiale du fichier JSON de données.

◆ generateCerfa()

App::Http::Controllers::PdfController::generateCerfa ( class Request $request)

Génère un formulaire CERFA 15497-03 pour les interventions sur équipements frigorifiques.

Cette méthode spécialisée traite les formulaires CERFA complexes avec :

  • Remplissage automatique des champs d'identification (opérateur, détenteur).
  • Gestion des types d'intervention (assemblage, maintenance, contrôle...).
  • Traitement des différents types de fluides frigorigènes (HCFC, HFC, HFO).
  • Gestion des périodicités de contrôle selon la réglementation.
  • Traitement des fuites détectées avec localisation et réparations.
  • Calcul des quantités de fluides chargés et récupérés.
  • Double signature numérique (opérateur et détenteur).
  • Formatage automatique des dates.
Paramètres
\Illuminate\Http\Request$requestRequête GET avec paramètres :
  • uid : Identifiant unique du formulaire
  • document : Type de document (cerfa_15497-03)
  • client : Identifiant du client
Renvoie
\Symfony\Component\HttpFoundation\BinaryFileResponse|\Illuminate\Http\JsonResponse Fichier PDF CERFA complété en affichage inline ou erreur JSON.
Exceptions

Illuminate\Http\Exceptions\HttpResponseException Si le fichier JSON de données n'existe pas.

Exceptions

InvalidArgumentException Si le format de date est incorrect.

Note
Conforme à la réglementation française sur les fluides frigorigènes.
Les coordonnées de positionnement sont précisément calibrées pour le formulaire CERFA officiel.
Deux signatures distinctes sont gérées : opérateur et détenteur d'équipement.
Avertissement
Les données doivent respecter le format attendu par l'administration française.
Voir également
https://www.service-public.fr/professionnels-entreprises/vosdroits/R14311 Documentation officielle CERFA.

◆ generateDownloadPDF()

App::Http::Controllers::PdfController::generateDownloadPDF ( class Request $request)

Génère et télécharge un PDF personnalisé à partir de données JSON.

Cette méthode traite les requêtes de génération de PDF pour différents types de documents :

  • Crée le répertoire de stockage si nécessaire.
  • Sauvegarde les données sous forme de fichier JSON.
  • Génère le PDF en remplissant les champs du modèle avec les données fournies.
  • Retourne le fichier PDF généré en téléchargement.

Types de documents supportés :

  • cerfa_15497-03 : Formulaire CERFA pour fluides frigorigènes
  • 1301-SD : Attestation TVA pour travaux
Paramètres
Request$requestRequête HTTP contenant les données JSON du formulaire. Doit contenir : uid, document, et les champs spécifiques au type.
Renvoie
mixed Téléchargement du PDF généré ou erreur JSON si le fichier source n'existe pas.
Exceptions
ExceptionSi une erreur survient lors de la création du fichier JSON.
ExceptionSi le modèle PDF source est introuvable.
Note
Le PDF généré est "aplati" pour empêcher les modifications ultérieures.
Avertissement
Cette méthode nécessite que les modèles PDF soient présents dans storage/app/public/.
Exemple:
POST avec JSON : {"uid": "12345", "document": "1301-SD", "nom": "Dupont", ...}

◆ show()

App::Http::Controllers::PdfController::show ( $token)

Affiche un document PDF à partir d'un token d'accès.

Cette méthode :

  • Recherche le fichier PDF correspondant au token fourni dans TokenLinksRapport.
  • Vérifie l'existence du fichier JSON associé contenant les métadonnées.
  • Extrait les informations du client, document et UID.
  • Retourne la vue d'affichage PDF avec les données nécessaires.
Paramètres
string$tokenJeton unique associé au document PDF pour l'identification sécurisée.
Renvoie
mixed Vue d'affichage du PDF ou erreur 404.
Exceptions
ExceptionSi aucun token valide n'est trouvé dans la base.
ExceptionSi le fichier JSON associé n'existe pas.
Note
Cette méthode est utilisée pour l'affichage dans le navigateur, pas pour le téléchargement direct.
Voir également
TokenLinksRapport Pour la gestion des tokens et leurs liens.
Exemple:
GET /pdf/show/abc123 pour afficher le PDF associé au token "abc123".

◆ upload()

App::Http::Controllers::PdfController::upload ( class Request $request)

Upload et traitement sécurisé de fichiers PDF pour les devis certifiés.

Cette méthode gère l'upload de fichiers PDF avec validation et organisation :

  • Validation stricte du format PDF uniquement.
  • Extraction et vérification du token depuis le nom de fichier.
  • Vérification de l'existence du token en base de données.
  • Organisation automatique des fichiers par organisation et devis.
  • Création automatique de l'arborescence de stockage.
  • Logging détaillé de toutes les opérations d'upload.
Paramètres
Request$requestRequête POST avec fichier :
  • pdf_file : Fichier PDF à uploader (obligatoire) Le nom du fichier doit contenir le token valide.
Renvoie
mixed Réponse JSON avec :
  • Succès : message, path, original_filename
  • Erreur : message d'erreur et code HTTP approprié
Exceptions
ExceptionSi le fichier n'est pas un PDF valide.
ExceptionSi le token n'existe pas en base.
ExceptionSi la création du répertoire ou le déplacement du fichier échoue.
Note
L'arborescence créée suit le pattern : {organisation_id}/devis/{devis_id}_{token}/
Tous les uploads sont tracés dans les logs pour audit et sécurité.
Avertissement
Seuls les fichiers PDF sont acceptés pour des raisons de sécurité.
Exemple:
POST /upload avec un fichier "ABC123.pdf" crée le chemin ORG001/devis/DEV456_ABC123/DEV456_ABC123.pdf

◆ viewDevis()

App::Http::Controllers::PdfController::viewDevis ( class Request $request,
$token )

Affiche la vue d'un devis PDF avec vérification du statut de certification.

Cette méthode de consultation sécurisée :

  • Vérifie l'existence et la validité du token de devis.
  • Détermine si une version certifiée du devis existe.
  • Prépare les données nécessaires pour l'affichage dans la vue.
  • Gère les accès non autorisés avec des erreurs 404 appropriées.
Paramètres
Request$requestRequête HTTP (généralement GET).
string$tokenToken unique d'identification du devis à consulter.
Renvoie
mixed Vue 'devis_pdf' avec les données du devis ou erreur 404.
Exceptions
ExceptionSi le token n'existe pas en base.
Note
La vue reçoit : client, token, nomDevis, isCertified pour l'affichage conditionnel.
La vérification de certification recherche un fichier avec suffixe '_certifie.pdf'.
Avertissement
Aucune vérification d'autorisation supplémentaire n'est effectuée au-delà de l'existence du token.
Exemple:
GET /devis/ABC123 affiche le devis associé au token "ABC123" avec son statut de certification.
